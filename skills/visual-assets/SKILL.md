# Visual Assets Skill

## Decision Tree — Which Tool For What?

```
What do you need to create?
│
├── A diagram, flowchart, architecture map, workflow, or relationship map?
│     └── → NOTEBOOKLM  (read notebooklm-infographics/SKILL.md)
│           Produces polished PNG from source content
│           Also see: .agents/workflows/notebooklm.md for step-by-step
│
├── An infographic from research, docs, or a data source?
│     └── → NOTEBOOKLM  (read infographic/SKILL.md)
│           Produces 4K PNG from actual source content
│           Takes 2–3 minutes via notebooklm CLI
│
├── A logo for a brand, product, or event?
│     └── → VISUALIZERSYSTEM SERVER  /api/generate-logo
│           5 style presets: flat, minimal, emblem, wordmark, photorealistic
│           Aspect ratio: 1:1 default
│
├── A hero banner, illustration, or creative image?
│     └── → VISUALIZERSYSTEM SERVER  /api/generate-image
│           OR generate_image tool (for quick concept)
│           Use the server for higher quality + save-to-disk
│
└── A slide deck or document with visuals?
      └── → PPTX SKILL (for decks) or DOCX SKILL (for documents)
            Embed images generated by the above tools
```

## Tool Reference

### 1. NotebookLM — Infographics, Diagrams, Workflow Maps
- **When:** Any visual map, workflow diagram, relationship diagram, architecture overview, infographic, or polished visual grounded in source content
- **How:** Via `notebooklm` CLI (see `.agents/workflows/notebooklm.md` for exact steps)
- **Output:** 4K PNG
- **Time:** 2–3 minutes per generation
- **Skill:** `notebooklm-infographics/SKILL.md` and `notebooklm/SKILL.md`
- **Cannot:** Edit existing images surgically — each generation is a full fresh render. Always describe the full layout you want.

### 2. VisualizerSystem Server — Images & Logos
- **Location:** `/Users/alethea/Documents/VisualizerSystem/server/`
- **Run:** `cd server && npm run dev` (port 4800)
- **UI:** `cd ui && npm run dev` (opens in browser)
- **Endpoints:**
  - `POST /api/generate-image` — general images
  - `POST /api/generate-logo` — logos with style presets
  - `GET  /api/health` — confirm server is running
  - `GET  /api/list-notebooks` — available NotebookLM notebooks
  - `POST /api/generate-infographic` — SSE stream, triggers notebooklm CLI

#### Logo Style Presets

| Style | Best For |
|-------|----------|
| `flat` | Startup brands, app icons, clean modern marks |
| `minimal` | Premium/luxury brands, consultancies, coaches |
| `emblem` | Sports teams, fraternal orgs, heritage brands |
| `wordmark` | Typography-forward brands, media companies |
| `photorealistic` | 3D product logos, tech/gaming brands |

### 3. generate_image Tool — Quick Concepts
- **When:** Fast concept art, UI mockups, quick design exploration
- **How:** Call `generate_image` with prompt and image name
- **Output:** Saved to artifacts directory as PNG
- **Best for:** Prototyping before committing to a full generation

## Combining Tools

Some tasks require multiple tools in sequence:

**Logo ON an infographic:**
1. Generate logo with VisualizerSystem (`/api/generate-logo`)
2. Describe logo placement in the NotebookLM infographic prompt
3. NotebookLM incorporates described visual elements into its composition

**Slides with custom assets:**
1. Generate hero image or logo via VisualizerSystem
2. Use PPTX skill to place images into slides at exact coordinates

## Theme Standards

All visual work should reference `creative-themes/SKILL.md` for approved color palettes. Default for AntiGravity brand work: **Warm Professional** theme (cream background `#faf5ee`, teal headers `#2c8c99`, gold accents `#d4a853`).

---

## Anti-Patterns

- ❌ Don't use generate_image when quality and saving-to-disk matters — use the server
- ❌ Don't ask NotebookLM to "just change one color" — it rerenders from scratch every time
- ❌ Don't guess at visual generation tools — check this decision tree first
