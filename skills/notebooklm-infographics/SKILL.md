# NotebookLM Infographics (Technical Architecture)

## ✅ Current System (Updated 2026-03-19)

Infographics are generated via **`nlm`** (`notebooklm-mcp-cli` v0.5.1), a unified CLI and MCP server. No browser agent, no Playwright, no Chromium is needed for any operation. The tool makes direct HTTP calls to NotebookLM's internal APIs using stored authentication cookies.

**Two access paths exist:**

1. **CLI (`nlm`)** — Run commands directly in the terminal. Best for this agent.
2. **MCP Server (`notebooklm-mcp`)** — 35 tools available to MCP-compatible AI agents. Configured in Antigravity's `mcp_config.json`. Available after session restart.

**Auth profile location:** `/Users/alethea/.notebooklm-mcp-cli/profiles/default`

## 🏗 The Infographic Pipeline

### Quick Path (Single Command)

```bash
nlm infographic create <notebook-id> --confirm \
  --orientation landscape \
  --detail detailed
```

This triggers generation, waits for completion, and returns the artifact ID. No need to manage steps manually.

### Full Pipeline (When You Need Control)

1. **Add Source (Optional)**
   ```bash
   nlm source add <notebook-id> --text "/path/to/source.md"
   ```

2. **Create Infographic**
   ```bash
   nlm infographic create <notebook-id> \
     --orientation landscape \
     --detail detailed \
     --confirm
   ```
   Returns artifact ID when generation completes (2-3 minutes).

3. **Download the PNG**
   ```bash
   nlm download infographic "/path/to/save/output.png" \
     --notebook <notebook-id> \
     --artifact <artifact-id>
   ```

### Alternative: Studio Commands

```bash
# Check all artifacts in a notebook
nlm studio status <notebook-id>

# Download by artifact ID
nlm download infographic ./output.png -a <artifact-id> --notebook <notebook-id>
```

## 🎨 Prompting Rules (No Surgical Edits)

**Never ask NotebookLM to "just change one color" or "move this box left."**

NotebookLM does **not** edit existing images. It re-renders a 100% brand new layout from scratch every single time. 

If the user likes a layout but wants the background color changed, you MUST explicitly describe the layout they liked in the new prompt (e.g., "Keep the 8-card grid layout, but make the background green"). If you don't describe the layout, NotebookLM will hallucinate a completely different structure.

- **Bad prompt:** "Make the background green this time."
- **Good prompt:** "Use an 8-card grid layout with icons. Teal headers, gold accents. Make the background of the example boxes dark forest green."

## 🔒 SECURITY — NEVER EXPOSE SENSITIVE IDENTIFIERS (Non-Negotiable)

**This rule is absolute. There are no exceptions. It applies to every infographic, diagram, image, or visual artifact generated by any tool in this workspace — NotebookLM, Gemini image generation, or any future tool.**

### What must NEVER appear in any generated visual:

- **Port numbers** (e.g., `4800`, `3001`, `5173`)
- **IP addresses** (e.g., `127.0.0.1`, `localhost`, internal IPs)
- **API keys, tokens, or secrets** of any kind
- **Internal file paths** that reveal system structure
- **Database connection strings or credentials**
- **Environment variable values**

### Why:

Infographics and diagrams are designed to be **shared** — on LinkedIn, in Skool, with clients, in presentations. A single port number or internal path baked into a visual asset is a security leak that cannot be recalled once posted. It tells attackers exactly where to probe.

### How to comply:

- **Before writing any prompt:** Mentally scan it for numbers that look like ports, paths that start with `/Users/` or `/home/`, and any string that resembles a key or token. Remove them.
- **Before feeding any source document:** If the source file (like a session ledger or README) contains port numbers or paths, either (a) create a sanitized copy first, or (b) explicitly instruct the prompt to "omit all port numbers, IP addresses, and file paths from the visual."
- **Use generic labels instead:** Say "VisualizerSystem Server" not "localhost:4800". Say "API Endpoint" not "http://127.0.0.1:4800/api/generate-image".

### Enforcement:

Any agent that puts a port number, IP address, or credential into a visual asset has violated this rule. This is not a guideline — it is a hard security boundary. Logged 2026-03-14 after a real incident.

## 🔧 Re-Authentication

If `nlm notebook list` fails with auth errors:

1. Run the old CLI to capture fresh cookies:
   ```bash
   /Users/alethea/.local/bin/notebooklm login
   ```
2. Convert and import:
   ```bash
   python3 -c "
   import json
   with open('/Users/alethea/.notebooklm/storage_state.json') as f:
       state = json.load(f)
   cookies = [{'name':c['name'],'value':c['value'],'domain':c['domain'],'path':c.get('path','/'),'secure':c.get('secure',True),'httpOnly':c.get('httpOnly',False)} for c in state['cookies'] if 'google' in c.get('domain','')]
   with open('/tmp/nlm_cookies.json','w') as f:
       json.dump(cookies, f)
   "
   nlm login --manual --file /tmp/nlm_cookies.json
   rm /tmp/nlm_cookies.json
   ```
3. Verify: `nlm notebook list`
