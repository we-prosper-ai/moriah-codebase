## Hardware Requirements (Hetzner CX32: 4 vCPU, 8GB RAM)

### Mail-in-a-Box
- **Minimum:** 1 vCPU, 512MB RAM  
- **Hetzner CX32 verdict:** ✅ Overkill — handles 20+ users easily  
- **Disk:** 10GB minimum (20-50GB recommended for mail storage)  

### Mailcow
- **Minimum:** 2 vCPU, 4GB RAM (8GB with ClamAV + SOGo)  
- **Hetzner CX32 verdict:** ✅ Perfect fit — full stack with all features  
- **User capacity:** 5-10 mailbox users, ~100 IMAP users total  
- **Note:** Can disable ClamAV/Solr/SOGo to reduce to 4GB, but not recommended  

### Modoboa
- **Minimum:** 1 vCPU, 2GB RAM  
- **Hetzner CX32 verdict:** ✅ Excellent — plenty of headroom  
- **User capacity:** 50+ users  

### iRedMail
- **Minimum:** 2 vCPU, 4GB RAM (8GB if SOGo groupware enabled)  
- **Hetzner CX32 verdict:** ✅ Very good — full stack including SOGo  
- **Note:** Recent ClamAV updates make 4GB tight; 8GB preferred  

### Postal
- **Minimum:** 1 vCPU, 2GB RAM  
- **Hetzner CX32 verdict:** ✅ Excellent — lightweight  
- **Note:** Transactional mail focus, not a full mailbox server

## Hetzner-Specific Considerations

⚠️ **Port 25 is blocked by default**
- Must request unblock after first invoice
- Takes 24-48 hours typically
- Critical for mail server functionality

✅ **Static IPv4 address** — Available on all plans  
✅ **Reverse DNS** — Fully configurable  
✅ **10 Gbps network** — Excellent for mail throughput

## Domain & DNS Setup (Required for All)

- Static public IP ✅
- Reverse DNS (rDNS) configured ✅
- MX record pointing to server ✅
- SPF/DKIM/DMARC records (auto-generated by Mail-in-a-Box & Mailcow; manual for others)
- TLS certificate (all auto-provision via Let's Encrypt)

## Backup Strategies

| Solution | Built-in | Recommendation |
|----------|----------|-----------------|
| Mail-in-a-Box | Duplicity (S3-compatible) | Use S3 + local snapshot |
| Mailcow | Manual (Docker volumes) | Docker volume backups + S3 sync |
| Modoboa | Manual | PostgreSQL dumps + file backups |
| iRedMail | Manual | Database dumps + mail folder backups |
| Postal | Manual | Database + config backups |

**For We Prosper AI:** Use Hetzner Spaces (S3-compatible) for automatic daily backups.

---

## Migration from Google Workspace

**Standard process (works with all 3 main solutions):**
1. Create users in target system
2. Use IMAP migration tool (Google Workspace data migration service)
3. Update MX records (72h propagation max)
4. Update SPF/DKIM/DMARC for new server
5. Verify deliverability

**Easiest options:**
- **Mail-in-a-Box:** Auto-generates SPF/DKIM/DMARC correctly
- **Mailcow:** REST API for bulk user creation
- **Modoboa:** Admin panel simplifies user setup

## Security & Compliance

### Encryption & TLS (All support)
- ✅ TLS 1.2+ enforced (Mail-in-a-Box: TLS 1.2+ only)
- ✅ Auto Let's Encrypt SSL/TLS provisioning & renewal
- ✅ SPF/DKIM/DMARC support
- ✅ DANE TLSA (Mail-in-a-Box with DNSSEC)
- ✅ MTA-STS (Mail-in-a-Box "enforce" mode; Mailcow added Sept 2025)

### GDPR Compliance
All offer data control on YOUR server (no third-party data sharing). **Important:** You're the data controller — responsible for:
- Data protection impact assessments (DPIAs)
- Retention policies & deletion procedures
- User right fulfillment (access, erasure, portability)
- Audit logging

### Audit Logging Support
- **Mail-in-a-Box:** Basic (logs in /var/log, health checks)
- **Mailcow:** Comprehensive (audit logs in web interface)
- **Modoboa:** Moderate (Django logging)
- **iRedMail:** Moderate (syslog integration)
- **Postal:** Full request/webhook history (7 days stored)

### 2FA Support (All have)
- Mail-in-a-Box: TOTP ✅
- Mailcow: TOTP, WebAuthn, Fail2ban-like protection ✅✅
- Modoboa: TOTP/HOTP ✅
- iRedMail: TOTP ✅
- Postal: API key management ✅

## Why NOT Postal for We Prosper AI

Postal is excellent **only if** you need:
- Transactional email delivery (like SendGrid)
- Programmatic send via REST API/SMTP
- Webhook event notifications

**Limitations for team email:**
- ❌ No user mailbox management
- ❌ No webmail
- ❌ No calendar/contacts
- ✅ Good for: API-driven architectures, tracking, multi-tenancy

## Alternative: Modoboa (If You Want Faster Setup)

Modoboa is your pick if you want to:
- Launch faster (~15 min vs ~45 min)
- Have simpler admin interface
- Still get calendar/contacts built-in
- Support same team size

Both Mailcow and Modoboa are excellent. Mailcow = more features + control. Modoboa = faster + simpler.
