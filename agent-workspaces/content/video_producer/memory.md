## Style Guide

### Core Principles
1. **Story-first approach** — Lead with narrative, not facts
2. **Conversational tone** — Like Tina's teaching style: warm, direct, authentic
3. **Clear structure** — Hook → Core concept → Example → Action step → CTA
4. **Optimal length** — 3-5 minutes (800-1200 words for performance)

### Script Format
```
[HOOK] (0:00-0:10)
- Open with a question, problem, or surprising insight
- Make viewers want to keep watching
- Reference the transformation available

[CORE CONCEPT] (0:10-2:30)
- Explain the main idea clearly
- Use simple language, avoid jargon
- Connect to viewer's life/business

[EXAMPLE] (2:30-3:30)
- Show a real case study or scenario
- Use specific numbers and details
- Make it relatable

[ACTION STEP] (3:30-4:30)
- Tell viewers exactly what to do next
- Be specific (not "learn more", but "download the checklist")
- Make it easy to take action

[CTA] (4:30-5:00)
- Clear next step (join community, buy course, watch next video)
- Build anticipation for what comes next
```

### Tone Guidelines
- Warm but direct (not overly casual)
- Confident (Tina knows her stuff)
- Transparent (admit limitations, share failures)
- Actionable (everything should serve the viewer)

## Quality Criteria — How to Know It's Done

| Criterion | How to Measure |
|-----------|----------------|
| Hook is compelling | "Would I watch this?" test |
| Concept is clear | A teenager could explain it back |
| Example is specific | Contains numbers, names, details |
| Action step is obvious | Viewer knows exactly what to do |
| Tone matches Tina | Reads like her voice, not AI-generic |
| Length is optimal | 800-1200 words (~4-5 minutes read time) |
| Structure flows | No jarring transitions |
| Grammar is clean | Professional-ready |
