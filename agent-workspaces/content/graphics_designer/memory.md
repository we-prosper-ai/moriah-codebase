## Style Guide

### Core Principles
1. **Accessibility first** — WCAG AA standard minimum, high contrast, readable fonts
2. **Consistent brand language** — Use Tina's visual identity (colors, typography, imagery style)
3. **Clarity over decoration** — Visual metaphors should clarify, not confuse
4. **Specification detail** — Hand off to contractors with complete clarity

### Design Brief Template

```
## [Visual Asset Name]

### Purpose
What problem does this design solve? Where will it appear?

### Key Message
What idea or emotion should this visual communicate?

### Audience
Who is viewing this? (beginners, entrepreneurs, technical founders)

### Style Elements
- Color palette: [Specific hex codes]
- Typography: [Font names and weights]
- Imagery style: [Photos, illustrations, graphs, etc.]
- Tone: [Professional, approachable, technical, inspiring]

### Technical Specs
- Format: [PNG, SVG, MP4, etc.]
- Dimensions: [1920x1080, etc.]
- File size target: [If applicable]
- Animation: [If applicable]

### Accessibility Requirements
- Alt text: [For images]
- Color contrast ratio: [Minimum 4.5:1 for text]
- Font size minimum: [12px for body, 18px for headings]
- Closed captions: [If video]

### Content Integration
How does this design fit with other visuals? What comes before/after?

### Delivery Checklist
- [ ] Final file in correct format
- [ ] Optimized for web/print as needed
- [ ] Accessibility review complete
- [ ] Brand consistency verified
- [ ] Approved by Tina
```

## Quality Criteria — How to Know It's Done

| Criterion | How to Measure |
|-----------|----------------|
| Brief is detailed | Contractor could execute without questions |
| Specs are precise | Dimensions, fonts, colors are exact |
| Accessibility is verified | All WCAG requirements met |
| Brand consistency | Matches existing visual language |
| Tone is clear | Designer knows the emotional intent |
| File optimization | Files are web-ready (compressed, optimized) |
| Delivery complete | All formats and variations provided |
